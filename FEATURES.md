# SOL-ACT 뮤즈 아카데미 - 전체 기능 문서

## 시스템 개요

연기 학원 통합 관리 플랫폼으로, 수업/과제/평가/포트폴리오/식단/소통 등을 하나의 앱에서 관리합니다.

- **프론트엔드**: React + TypeScript (Vite)
- **백엔드**: FastAPI (Python)
- **데이터베이스**: SQLite
- **실시간 통신**: WebSocket (채팅, 알림, 데이터 동기화)
- **인증**: JWT 토큰 (자동 로그인 지원)
- **PWA**: 설치형 앱, 오프라인 캐싱, 웹 푸시 알림

### 계정 역할

| 역할 | 설명 | 데이터 범위 |
|------|------|------------|
| **원장 (Director)** | 학원 전체 관리자 | 모든 데이터 접근 가능 |
| **선생님 (Teacher)** | 수업 담당 강사 | 자기 반 학생 데이터만 접근 |
| **수강생 (Student)** | 학원 수강생 | 본인 데이터만 접근 |

## 1. 인증 및 계정 관리

### 1-1. 로그인
- 이메일 + 비밀번호 로그인
- 자동 로그인 (토큰 저장)
- 데모 모드 지원 (체험용 계정)

### 1-2. 회원가입
- **초대 코드 필수**: 원장이 생성한 8자리 코드 (형식: XXXX-XXXX)
- 코드에 역할 정보 포함 (수강생/선생님)
- 1회 사용 후 자동 만료
- 비밀번호 규칙: 8자 이상, 영문+숫자+특수문자 포함

### 1-3. 아이디/비밀번호 찾기
- **아이디 찾기**: 이름으로 검색 → 마스킹된 이메일 표시
- **비밀번호 찾기**: 이메일+이름 확인 → 임시 비밀번호 발급

### 1-4. 프로필 설정
- 이름, 이메일 변경
- 프로필 사진 업로드
- 비밀번호 변경 (실시간 유효성 검사 UI)
- 푸시 알림 설정 (켜기/끄기, 상태 확인)

## 2. 클래스(반) 관리

### 기능
- 반 생성: 이름, 설명, 과목+선생님 배정, 주간 스케줄
- 주간 스케줄: 요일 + 시작/종료 시간 (예: 월 14:00~16:00)
- **자동 수업 생성**: 스케줄 기반으로 향후 4주 수업 자동 생성

## 3. 수업 관리

### 기능
- **수업 유형**: 정규 수업 / 개인 레슨
- **수업 상태**: 예정(Scheduled) → 완료(Completed) / 취소(Cancelled)
- 수업 정보: 날짜, 시간, 장소, 과목, 담당 선생님, 메모
- **일괄 수업 생성**: 날짜 범위 + 요일 선택으로 반복 수업 생성
- 수업 취소/완료 처리 시 학생에게 알림
- 출석 관리 (수업별 학생 출석 체크)

### 수업 일지
- **일지 유형**: 선생님 일지 / 학생 일지
- 내용, 수업 목표, 다음 수업 계획, 미디어 첨부
- **AI 피드백**: 일지 내용 기반 AI 코칭 피드백 생성
- 작성 시 상대방에게 알림 (학생→선생님, 선생님→학생들)

### 출석
- **출석 상태**: 출석 / 지각 / 결석 / 사유 결석
- **일괄 출석 처리**: 수업별 전체 학생 한번에 체크
- 출석 통계: 학생별/반별 출석률 계산

## 4. 과제 관리

### 기능
- 과제 생성: 제목, 설명, 마감일
- **일괄 과제 부여**: 개별 학생 / 반 전체 선택 가능
- **과제 상태**: 미제출(Pending) → 제출완료(Submitted) → 채점완료(Graded)
- 제출: 텍스트 입력 + 파일 첨부 (영상/문서)
- **채점**: 등급 선택 (S/A/B/C/D) + 피드백 작성
- **AI 분석**: 독백 제출물에 대한 AI 코칭 분석
- **캘린더 뷰**: 마감일 기준 달력 표시 + 목록 보기 전환

### 프론트엔드 UI
- 달력 보기 / 목록 보기 토글
- 학생 필터 드롭다운 (선생님용: 자기 반 학생만 표시)
- 선생님 필터 드롭다운 (원장용: 전체 선생님)
- 과제 생성 모달: **"개별 학생" / "반 전체"** 탭 전환

### API 엔드포인트

## 18. 데이터 범위 제한 (Data Scoping)

### 핵심 원칙
> **선생님은 자기 반 학생 데이터만, 원장은 전체 데이터 접근**

### 백엔드 적용 범위

| 엔드포인트 | 선생님 제한 | 원장 |
|-----------|------------|------|
| `GET /api/users/` | 자기 반 학생 + 본인만 | 전체 |
| `GET /api/classes/` | 담당 반만 | 전체 |
| `GET /api/assignments/` | 담당 학생 과제만 | 전체 |
| `GET /api/evaluations/` | 담당 학생 평가만 | 전체 |
| `GET /api/portfolios/` | 담당 학생 포트폴리오만 | 전체 |
| `GET /api/diet/` | 담당 학생 식단만 | 전체 |
| `GET /api/lessons/` | 담당 반 수업만 | 전체 |
| `GET /api/journals/` | 담당 반 수업 일지만 | 전체 |

### 생성 시 검증
- 선생님이 과제 부여 → 자기 반 학생에게만 가능
- 선생님이 평가 작성 → 자기 반 학생만 가능
- 선생님이 수업 생성 → 자기 반에만 가능

### 프론트엔드 자동 적용
백엔드에서 사용자 목록을 범위 제한하므로, 프론트엔드의 모든 학생 드롭다운/선택 UI가 자동으로 해당 선생님의 반 학생만 표시합니다.
