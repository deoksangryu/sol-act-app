# 쏠연기뮤지컬학원 모바일 앱 프로토타입 — v8 명세서

> 다른 AI 에이전트나 개발자가 이 프로토타입을 이어서 작업할 때 참고하는 문서입니다.

## 1. 프로젝트 개요

**대상**: 쏠연기뮤지컬학원(부평) — 입시 연기·뮤지컬 학원
**사용자**: 입시생(연극영화·뮤지컬·자유무용 진학 준비), 강사, 학원장
**도메인**: 한국 예체능 입시 — 자유연기 독백, 발성, 뮤지컬 넘버, 자유무용
**스택**: 순수 HTML + JavaScript (프레임워크 없음), Tabler Icons, 자체 SVG 아이콘

**프로토타입 목표**:
1. 토스(Toss) 디자인 원칙을 학원 운영 도메인에 적용
2. 학생·선생님 양방향 워크플로우의 핵심 흐름 검증
3. 푸시 알림이 매개하는 비동기 협업 모델 구현
4. 수업 카테고리(연기·뮤지컬·무용)의 시각적 정체성 확립

## 2. 적용한 토스 디자인 원칙

### Product Principles
- **1 thing per 1 page** — 한 화면 = 한 가지 핵심 행동
- **Casual Concept** — 친근한 톤, "확인" → "네 제출할게요"
- **Minimum Features** — 부가 기능 줄이고 핵심 흐름에 집중
- **Clear CTA** — 큰 파란색 하단 고정 버튼
- **Value first, Cost later** — 가치 보여주고 입력은 나중에
- **Easy to answer** — 칩 선택 위주, 자유 입력 최소화

### UX Writing 8원칙 적용
- 해요체, 능동형, 긍정형
- "취소" → "닫기", "확인" → "네 ~할게요"
- 격려/감사 톤: "오늘도 와줘서 좋아요"
- 안내 메시지: 큰 글자(21px 700) + 두 줄 줄바꿈

### 인사이트
> A/B 테스트에서 **단계 수 감소보다 맥락 설명이 전환율의 핵심 레버**다.

## 4. 수업 카테고리 (v8 핵심 결정)

학원의 수업을 세 가지 카테고리로 단순화하고, 각각에 시각적 정체성을 부여:

| 카테고리 | 키 | 색상 | 아이콘 메타포 | 파일 |
|---|---|---|---|---|
| 연기 | `acting` | BLUE `#3182F6` | 페르소나 마스크 (윤곽 + 눈 + 미소) | `icon-acting.svg` |
| 뮤지컬 | `musical` | PURPLE `#6D5BD0` | 마이크 + 작은 음표 | `icon-musical.svg` |
| 무용 | `dance` | PINK `#E84F8B` | 아라베스크 동작 인물 | `icon-dance.svg` |

세 아이콘 모두 100×100 viewBox, stroke-width 5~7, round line-cap/join으로 일관성 유지. SVG 함수 `SVGIc[cat](color, size)`로 동적 색상 주입.

`catC(cat)` 헬퍼로 카테고리 → 색상 페어 반환:
```javascript
function catC(c) {
  if (c === 'acting')  return {bg: BB,     fg: B};
  if (c === 'musical') return {bg: PB,     fg: P};
  if (c === 'dance')   return {bg: PINKBG, fg: PINK};
  return {bg: SF, fg: S};
}
```

수업 세션 데이터는 `cat` 필드로 카테고리를 명시:
```javascript
{i:'c5', c:'독백 연기 클래스', cat:'acting', dy:'11/18 화', tm:'오후 3:00', w:'today'}
```

## 5. 화면 구조

### 하단 탭 (5개)
| ID | 이름 | 학생 관점 | 선생님 관점 |
|---|---|---|---|
| `cs` | 수업 | 출석 + 일지 작성 | 출결 + 수업일지 + 학생 일지 |
| `tk` | 과제 | 받은 과제 / 제출 | 출제 + 제출 현황 |
| `vd` | 영상 | 연습 영상 업로드 / 피드백 확인 | 학생 영상 피드백 |
| `bd` | 식단 | 체중 + 식단 기록 | 학생 체중/식단 + 피드백 |
| `mu` | 음악 | 무용음악 청취 + 다운로드 요청 | 라이브러리 + 요청 승인/거절 |

### 상단 영역
- 상태바: 시간, 시그널, WiFi, 배터리 + **벨 아이콘** (안 읽은 알림 뱃지)
- 역할 전환 토글: 학생 ↔ 선생님

### 토스트 알림
- 역할 전환 시 새 역할의 최신 안 읽은 알림이 검은 배너로 슬라이드 다운
- 4.5초 후 자동 사라짐, 누르면 해당 화면으로 이동

## 6. 핵심 디자인 결정 (대화 중 확정)

이 결정들은 **사용자(학원 운영 도메인 전문가)와의 반복**을 거쳐 확정됐습니다. 변경 시 사용자에게 다시 확인하는 것을 권장합니다.

1. **일정과 일지를 통합** → "수업" 탭 하나로
2. **일지는 수업과 1:1로 묶임** — 자유 일지 없음, 수업 선택 단계 생략
3. **선생님 수업일지는 학생 비공개** — 클래스 단위 운영 기록, 댓글 없음
4. **선생님 → 학생 코칭은 댓글/피드백으로만** — 선생님이 학생 개인 일지를 쓰지 않음
5. **음악 다운로드 거절 시에도 인앱 청취 유지** — 저작권 보호 + 학생 학습권 균형
6. **자유무용 입시 음원이 핵심 사용 사례** — 사용 목적 칩: 자유무용 입시 연습 / 워크숍 / 콩쿠르 / 수업 복습
7. **수업 카테고리 = 연기 · 뮤지컬 · 무용** (v8 신규) — 다른 분류 추가 시 SVG 아이콘과 색상 톤도 함께 설계

## 7. 푸시 알림 시스템

### 7.1 UI 구성
- 벨 아이콘 (상태바 우측) + 빨간 뱃지
- 토스트 배너 (역할 전환 시)
- 알림 목록 화면

### 7.2 자동 트리거 매트릭스

**학생 → 선생님**
| 학생 행동 | 알림 제목 | 타깃 화면 | 톤 |
|---|---|---|---|
| 일지 작성 | `<이름>님이 일지를 남겼어요` | `sv:cls\|sid` | l |
| 과제 제출 | `<이름>님이 과제를 제출했어요` | `tv:taskId` | t |
| 영상 업로드 | `<이름>님이 영상을 올렸어요` | `vv:vidId` | v |
| 식단 업로드 | `<이름>님이 식단을 올렸어요` | `mv:mealId` | m |
| 음원 요청 | `<이름>님이 음원 요청을 보냈어요` | `mm:trackId\|sid` | u |

**선생님 → 학생** (대상이 본인 페르소나일 때만 푸시)
| 선생님 행동 | 알림 제목 | 타깃 화면 | 톤 |
|---|---|---|---|
| 일지 댓글 | `선생님이 일지에 댓글을 남겼어요` | `sv:cls\|sid` | l |
| 새 과제 | `새 과제가 도착했어요` | `tv:taskId` | t |
| 영상 피드백 | `영상 피드백이 도착했어요` | `vv:vidId` | v |
| 식단 피드백 | `식단 피드백이 도착했어요` | `mv:mealId` | m |
| 음원 승인 | `음원 다운로드가 승인됐어요` | `mp:trackId` | u |
| 음원 거절 | `음원 요청이 검토됐어요` | `mp:trackId` | u |

톤 코드: `l`=일지, `t`=과제, `v`=영상, `m`=식단, `u`=음악

## 9. SVG 아이콘 시스템

### 사용 방식
```javascript
// 카테고리 키로 아이콘 생성 (색상과 크기 주입)
SVGIc.acting(color, size)
SVGIc.musical(color, size)
SVGIc.dance(color, size)
```

### `ic()` 함수
Tabler 아이콘과 SVG 아이콘을 모두 지원:
- 이름이 `'ti-'`로 시작하면 → Tabler 웹폰트
- 이름이 SVGIc 키와 일치하면 → SVG 함수 호출
- 둘 다 아니면 빈 컨테이너

```javascript
ic('acting', BB, B, 44)      // 연기 아이콘, 파랑 배경, 파랑 stroke
ic('ti-school', BB, B, 44)   // Tabler 학교 아이콘
```

### 파일 형태
세 아이콘은 standalone SVG 파일로도 제공:
- `icon-acting.svg`
- `icon-musical.svg`
- `icon-dance.svg`

각 파일은 자체적으로 색상이 박혀 있어서(BLUE/PURPLE/PINK) 디자인 도구에서 바로 사용 가능. 코드에서 동적으로 쓸 때는 `SVGIc` 함수가 색상을 주입.

## 12. 현재 단순화된 부분 (도구 안정성을 위해)

- **체중 추이 그래프**: SVG 꺾은선 그래프 → 텍스트 리스트

## 13. 확장 후보 (우선순위 순)

1. **홈 / 알림 화면** — 오늘 할 일 통합 뷰
2. **수업 시간표 캘린더** (월/주 뷰)
3. **학부모 권한** — 자녀 정보 열람
4. **공지/푸시 채널** — 학원 공지, 시즌 안내
5. **결제 / 수강료**
6. **포트폴리오 / 입시 진척도** — 지원 학교, 합격 상태
7. **메시지 / 채팅** — 일지 댓글 외 직접 대화
8. **신체 기록 확장** — 키, 부상, 컨디션 일지
9. **시즌별 콘텐츠** — 입시 시즌(11-12월) 대응 UI
10. **수업 카테고리 추가 가능성** — 만약 학원에서 새 분야(예: 매체연기, 보컬 등) 추가 시 SVGIc에 새 함수 + 색상 토큰 + catC 추가 (현재 PINK 다음은 GREEN/ORANGE 추천)

## 14. 도메인 컨텍스트 (한국 예체능 입시)

- **자유연기**: 학생이 선택한 대본 3분 분량 독백
- **지정연기**: 학교별 지정 대본 (매년 발표)
- **자유무용**: 음악 선택 + 안무 직접 구성 → 입시생이 음원 다운로드 필요
- **자유곡 (뮤지컬)**: 뮤지컬 넘버 선택
- **수시 입시 시즌**: 9-11월, **정시**: 1-2월
- **학원 규모**: 약 7-19명 (시즌별 변동)
- **수업 종류**: 연기(발성·호흡·독백·그룹 장면), 뮤지컬(넘버·연기 결합), 무용(자유무용·기본 동작)

## 15. 기술 노트

- **상태 관리**: 단일 `D` 객체, 가변 (no immutability)
- **렌더링**: `dr()` 함수가 전체 다시 렌더 — jQuery-style
- **이벤트**: `data-*` 속성 + 이벤트 위임 (`closest()`)
- **토스트 애니메이션**: setTimeout(20ms) 트릭 + CSS transition
- **에러 핸들링**: 모든 핸들러를 try/catch로 감싸 화면이 죽지 않도록
- **SVG 아이콘**: 동적 색상 주입 — 호출 시점에 `fill`/`stroke`에 색 삽입

## 17. v7 → v8 변경사항

- **수업 카테고리 신설**: 연기 · 뮤지컬 · 무용 (이전엔 자유 텍스트)
- **SVG 아이콘 시스템 추가**: `SVGIc` 객체 + 카테고리별 SVG 함수
- **PINK 색상 토큰 신규**: `#E84F8B` / `#FCE7F0`
- **`catC()` 헬퍼**: 카테고리 → 색상 페어
- **`ic()` 함수 확장**: Tabler + SVG 둘 다 지원
- **수업 세션 데이터**: `c.ic`(Tabler 아이콘 이름) → `c.cat`(카테고리 키)
- **수업 데이터 5개로 확장**: 연기 2 + 무용 2 + 뮤지컬 1 (분포 균형)
